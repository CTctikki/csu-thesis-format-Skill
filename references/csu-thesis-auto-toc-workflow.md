# 中南大学论文自动目录工作流

这份文档只处理一类问题：`目录看起来像目录，但本质上不是可更新的 Word 自动目录`。

适用场景：

- 目录是手打文本，`TablesOfContents.Count = 0` 或 `Fields.Count = 0`
- 目录页码漂移，正文第 1 章页码不再从 1 开始
- 目录重复，通常是旧静态目录和新 TOC 同时存在
- 需要把当前论文改成“后续还能继续更新”的目录

## 1. 先决定交付模式

1. `继续修改中的论文`
- 优先保留真正的 TOC 域，后续插图、改标题、扩写正文后还能更新目录。

2. `交导师前的 Word 定稿`
- 先在 Word 中把 TOC 更新到最新，再决定是否保留为自动目录。
- 如果导师后面还会继续修改正文，保留自动目录更方便。

3. `追求最稳定展示的最终交付`
- 先在 Word 中更新 TOC，再导出 PDF。
- 如果第三方预览器经常把页码渲染成 `_docxField:PAGE_`，以 Word/PDF 为准，不把预览器当唯一判断标准。

## 2. 自动目录的前提

- 正文标题必须是真正的标题层级，至少要有 `Heading/标题 1/2/3` 或明确的大纲级别。
- 不要把“题目”“摘要”“ABSTRACT”误挂到正文目录层级里，除非你就是想把它们放进目录。
- 目录和正文之间要有明确的 `section` 边界。
- 正文页码要从第 1 章重新开始；前置部分按学校规则使用罗马页码或单独编号。

## 3. 推荐修法

1. 先复制出新版本，不在原件上直接硬改。
2. 找到 `目录` 标题段落，以及正文里第一个真正的章节标题，通常是 `第1章 ...`。
3. 如果 `目录` 和正文第 1 章之间已经塞了一整段手打目录，把这些旧条目先删掉。
4. 检查 `目录` 后到正文前是否还保留正确的分页/分节。
- 常见目标是：目录结束后开始新节，正文页码从 `1` 重新计。
5. 只给正文里的真实标题设置目录层级。
- 章标题：`第1章`、`第2章`、`附录A`、`致谢`、`参考文献`
- 节标题：`1.1`、`2.3`
- 小节标题：`2.2.1`
6. 在 Word 中插入或更新 TOC。
- 不要一边保留旧静态目录，一边再插入新的 TOC 域。
7. 统一调整 `TOC 1 / TOC 2 / TOC 3` 样式。
- 当前这套中南大学定稿里，目录条目常见可交稿值是 `10.5 pt` 或 `12 pt`
- `TOC2` 常保留两个汉字左右缩进
- 页码通常放在右侧，配点引导线
8. 在 Word 里做最后一次字段更新。
- 常用动作是 `Ctrl+A` 后按 `F9`
9. 更新完目录后，再看一次正文第 1 章是否仍从 `1` 开始。

## 4. 验收清单

- `目录` 标题存在，且居中
- DOCX 内至少存在 1 个 `TOC` 域
- 目录里没有重复的 `第1章`
- 正文第 1 章页码从 `1` 开始
- 目录页码和正文位置一致
- 目录最好仍保持单页；如果被挤成两页，优先微调 TOC 样式，不要乱动正文

## 5. 常见失败模式

### 5.1 目录看着正常，但根本不是自动目录

症状：
- `目录` 下方是很多普通段落
- `TablesOfContents.Count = 0`
- `Fields.Count = 0`

说明：
- 这通常就是手打目录或旧静态目录，不具备更新能力。

### 5.2 自动目录插入后，正文第 1 章变成第 2 页

根因：
- 目录和正文之间原本负责重启页码的 `section break` 被删掉了。

修法：
- 恢复目录后的新节
- 让正文节的页码重新从 `1` 开始

### 5.3 目录重复两遍

根因：
- 旧静态目录没删，又插入了新的 TOC。

修法：
- 保留一套即可，通常删旧静态目录，再更新新 TOC。

### 5.4 预览里出现 `_docxField:PAGE_`

根因：
- 预览器不认 Word 域或字段缓存。

修法：
- 以 Word 打开效果和 Word 导出的 PDF 为准。
- 不要仅因为预览器异常，就误判 Word 目录本身是错的。

### 5.5 目录项变蓝或带下划线

根因：
- `Hyperlink` 或相关字符样式还带着链接格式。

修法：
- 检查 `TOC` 样式、`Hyperlink` 样式和标题字符样式，不只改单个 run。

## 6. 和现有脚本怎么配合

先跑：

```bash
python scripts/extract_docx_format.py thesis.docx
python scripts/check_csu_thesis_docx.py thesis.docx
python scripts/audit_csu_word_structures.py thesis.docx
```

重点看：

- `check_csu_thesis_docx.py`
  是否提示“目录更像手打/静态目录，不是可更新 TOC”

- `audit_csu_word_structures.py`
  是否存在 TOC 域、目录条目是否重复、目录前后是否有明显结构异常

如果脚本已经提示目录是静态的，就按这份文档把它转成真正的自动目录，再继续后面的页眉、页脚、公式和标题样式修订。
