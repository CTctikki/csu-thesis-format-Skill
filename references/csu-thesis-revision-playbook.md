# 中南大学论文真实修订流程

这份 playbook 只沉淀已经被真实论文修订验证过的流程，目标不是“生成一篇论文”，而是把一份快交稿的 Word 论文拉回模板要求。

## 1. 先分清三类问题

1. `纯文本/内容问题`
- 说明文字、图题表题、参考文献条目、公式变量释义、旧口径数字。

2. `Word 结构问题`
- section、页码、目录域、页眉页脚、标题样式链、字段更新、只读锁、页眉图片关系。

3. `渲染差异问题`
- Word 里看着对，预览里不对；或者 PDF 对、DOCX 预览不对。

不要把第 2、3 类问题当成“改几个字体和空格”。

## 2. 固定修订顺序

1. 复制出新版本文件，保留上一个版本。
2. 提取结构和样式：
```bash
python scripts/extract_docx_format.py thesis.docx
python scripts/check_csu_thesis_docx.py thesis.docx
```
3. 先改正文内容、图表题、参考文献、说明性文字。
4. 再改 section 骨架：封面、中文摘要、英文摘要、目录、正文分页与分节。
5. 再改页眉页脚、罗马/阿拉伯页码、首页不同、Link to Previous。
6. 再改标题样式、标题颜色链、图表编号、公式编号。
7. 再统一正文引用上标、特征下标和指数上标。
8. 最后才做自动目录更新、页码刷新、静态定稿。
9. 导出并检查关键页。

## 3. 定稿模式建议

### 草稿修订模式
- 允许保留目录占位说明。
- 允许标题样式先不完全正规化。
- 重点是内容修订和主要分页不乱。

### 交导师定稿模式
- 目录必须最终定型，不要把“等导师电脑更新目录”留给对方。
- 标题颜色必须显式锁黑，避免不同渲染器吃到主题蓝。
- 页眉页脚、页码、目录页码、正文首页都要在 Word 中确认。
- 公式必须统一成一种编号方案。

## 4. 目录处理规则

1. 改标题、插图表、扩写正文之前，不要频繁强依赖自动目录。
2. 进入最终定稿时，再统一插入或更新自动目录。
3. 若最终交付目标是“稳定打开即正确”，优先把目录 `Update` 后再固化成静态文本。
4. 目录重复通常不是显示问题，而是文档里同时存在：
   - 旧静态目录段落
   - 新 TOC 域展开后的目录段落
5. 目录条目不要只按模板默认样式机械回刷。若当前定稿版本已经人工调成单页、小四号、`TOC2` 两个汉字缩进，并且视觉正确，优先保留该定稿样式。

## 5. 页眉页脚规则

- 页眉页脚跟 `section` 绑定，不跟单页绑定。
- 先断开 `Link to Previous`，再分别改摘要页、目录页、正文页。
- 学长论文那种页眉，本质上通常是：
  - 左侧校徽校名图
  - 右侧题目
  - 下一段下划线空格或横线
- 如果只想稳定复刻观感，优先用“图 + 题目”结构，不要在兼容模式下过度依赖复杂浮动对象。

## 6. 公式规则

- 先确定唯一编号方案。推荐按章编号，如 `(2-1)`。
- 纯文本伪公式常见症状：
  - `Word OMaths.Count = 0`
  - 公式只是正文段落，编号靠空格硬推到右边
- 如果短时间内不追求全部转成专业公式对象，可以先统一成：
  - 单独一行
  - 居中公式本体
  - 行末独立编号
  - 编号连续
- 但如果已经进入定稿阶段，或前一轮编辑把公式改坏了，优先恢复为真正的 Word 公式对象，具体方法见 `references/csu-thesis-word-equation-objects.md`。
- 已验证的稳定路线是：
  1. 复制出新版本。
  2. 用 Word COM 打开副本。
  3. 保留“左空白 + 中间公式本体 + 右侧编号”的三列表格结构。
  4. 往公式单元格写入 Word 线性公式语法。
  5. 调用 `OMaths.Add(range)` 和 `BuildUp()` 生成公式对象。
  6. 长公式拆成多行时，编号只放最后一行行末。
- 不要再用 `python-docx` 的 `paragraph.text` 或 `cell.text` 去覆盖展示公式正文；这很容易把原有公式区域压扁成普通文本。
- 变量释义和“式中：”说明保持普通正文段落，不混进公式编号链。

## 6A. 参考文献规则补充

- 不要只凭第一段 run 的直接格式判断参考文献字体。
- 正文中的文献引用应是上标 run，例如 `[1]`、`[1-3]`、`[14,15]`；参考文献列表条目前的 `[1]`、`[2]` 不能改成上标。
- 可用 `scripts/normalize_citations_and_features.py input.docx output.docx` 自动处理正文引用上标，并跳过 `参考文献` 标题之后的文献条目。
- 中南大学这套模板里，参考文献条目常由 `书目1` 样式统一控制：
  - `楷体_GB2312 + Times New Roman`
  - 五号，约 `10.5 pt`
  - 悬挂缩进
- 因此，脚本和人工检查都应优先看样式链，而不是只看单个 run 的局部属性。

## 6B. 特征符号上下标规则

- 机器学习特征符号应全篇统一，不要只让某个表格内正确。正文、图题表题、表格表头和公式说明中都应使用相同口径。
- 常见写法：`d_WC`、`f_WC`、`x_Ni`、`x_Co`、`a_WC`、`A_sp`、`T_norm`，其中 `WC/Ni/Co/sp/norm` 是下标。
- 指数和单位幂使用上标，例如 `f_WC^(2/3)`、`R^2`、`cm^-3`、`μm^-1`、`mm^2/s`。
- 可用 `scripts/normalize_citations_and_features.py input.docx output.docx` 处理 `dWC/fWC/xNi/aWC/Asp/Tnorm/fWC2/3/R2/cm-3` 等紧凑写法，也可继续用 `scripts/unify_inline_subscripts.py` 处理带下划线的旧写法。

## 7. 标题颜色链规则

- Word 标题颜色不只来自 run 直接格式。
- 要同时检查：
  - `theme1.xml`
  - `styles.xml` 中的 `Heading 1/2/...`
  - 链接字符样式，例如“标题 1 字符”“标题 2 字符”
  - 段落或 run 直接格式
- 如果学校要求黑色标题，最终应在样式层显式写黑，不留 `accent1` 之类主题色。

## 8. 文件锁与 COM 处理

- 遇到“只读”“Permission denied”“WINWORD 无标题后台进程”时：
  1. 先检查是否有后台 `WINWORD`
  2. 关掉无窗口进程
  3. 必要时另存新文件继续
- 不要在锁住的原件上反复重试保存。

## 9. 最终人工检查页

至少检查：
- 封面
- 中文摘要页
- 英文摘要页
- 目录页
- 正文首页
- 第一处公式页
- 图表密集页
- 参考文献首页与末页

这些页面最容易被导师第一眼挑出问题。
